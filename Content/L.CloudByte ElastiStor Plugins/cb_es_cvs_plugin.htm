CloudByte Virtualization Service plug-in for VMware VCenter

Introduction

CloudByte Virtualization Service (CVS) plug-in is an array based backup, recovery, and cloning solution for virtual machines on CloudByte ElastiStor. The plug-in integrates CloudByte ElastiStor Storage services with VMware technologies and provides a single console to manage the virtual infrastructure. With CVS, administrators can monitor and manage storage to improve performance and availability directly from the VMware vCenter console instead of having to separately manage the storage systems.

The plug-in can direct VMware virtual machine level backup, clone, provision, and recovery operation using snapshot, restore, clone, and QoS capabilities of CloudByte ElastiStor and thus providing an end-to-end storage monitoring and management for VMware infrastructures.

For a list of operations supported by the CVS plugin, see the section "Setting up the storage"

Prerequisites

Modify configuration files

  1. SSH to the ElastiStor Node and log in as root.
  2. Do the following with the configuration file /boot/loader.conf
    1. Open the file using the command vi /boot/loader.conf
    2. Add the following line: vfs.nfsd.enable_cvs=1
    3. Save and exit using the command :wq
  3. Do the following with the configuration file /etc/sysctl.conf
    1. Open the file using the command vi /etc/sysctl.conf
    2. Add the following line in the configuration file: vfs.nfsd.enable_cvs=1
    3. Save and exit using the command :wq
  4. Reboot the Node using the command reboot.

The following topics explain how to set up and use the CVS plug-in.

Setting up the plug-in

The setup involves a Windows 2008 R2 machine with Java 6.0, vCenter Server 5.0, and CVS plug-in installed.

On vCenter (CVS Server)

  1. Get the installation files from CloudByte Technical Support.
  2. Double-click the setup.exe. The CloudByte Virtualization Services wizard appears.
  3. Follow the onscreen instruction and complete the wizard.
    • (Optional) To create a desktop icon of the plug-in, check Create desktop shortcut icon option in the wizard.
    • (Optional) To automatically launch the plug-in after installation, check Launch CloudByte Virtualization Service option in the wizard.

Setting up the storage

The storage can be set up either through the GUI or using CloudByte ElastiStor REST APIs.

CVS plug-in Graphical User Interface

The CVS plug-in user interface acts as a reference for consuming CloudByte Virtualization Services APIs. Administrators can refer to the UI code and implement similar functionality for their portals.

Note:This document uses the term TSM to imply Virtual Storage Machine (VSM).TSM has been renamed as VSM. However, changes are yet to take effect in the plug-in UI.

Launching the GUI

Launch the CVS GUI on the browser by specifying the vCenter IP address in the format as shown in the following example: endPoint=http://20.10.81.200:8000/cvsui/

CVS API reference

The CVS REST APIs provide a bundle of functions that encompasses all the features in the management console of CVS. The API provides access to the VMware infrastructure components (datacenters, datastores, virtual machines).

Forming the API request

Base URL : http://<IP address of the CVS server>:8000/cvs/

Method Web service command Description Parameters
POST datastore/create Creates a VMware datastore.
  • datacenter: vCenter datacenter where datastore has to be added.
  • host:IP address of ESX where datastore has to be added.
  • tsm: Name of the TSM in which the Storage Volume has to be added.
  • datastore: Name for the new datastore.
  • size: Size of the datastore.
  • iops: IOPS the user wants to set on datastore.
  • type: Type of VMware DataStore. Specify n for NFS or v for VMFS.
  • blocksize: Blocksize the user wants to set on datastore
GET vcutils/datastore/list Provides a list of datastores.

This operation does not require a request body.
POST datastore/delete Deletes a datastore.
  • datastore: Datastore that must be deleted.
POST datastore/edit Modify QoS parameters for datastore.
  • datastore: Specify the name of the datastore to be modified.
  • iops:The new IOPS value the user wants to set on datastore.
  • blocksize:The new blocksize that the user wants to set on datastore.
POST create/vm Creates a virtual machine.
  • dc: Specify the datacenter name in which the virtual machine has to be created.
  • ds:Specify the datastore name in which virtual machine has to be created.
  • guestosid:specify id of the OS in which the VM has to be created.
  • vm: Specify a name for the virtual machine to be created.
  • cpucount: Specify the number of CPUs to be added to the virtual machine.
  • memsize: Specify RAM size (only in MB) for the virtual machine to be created.
  • disksize: Size of OS disk on which the virtual machine has to be created.
  • net: specify virtual machine network name. Example: VMNetwork
  • nic: Specify name of the NIC to which the TSM is attached.
  • iops: IOPS the user wants to set on OS disk.
  • recordsize: Specify record size of OS disk.
GET vcutils/vm/list?istemplate=n List Virtual Machine This operation does not require a request body.
POST clone/delete Delete Virtual Machine
  • template:Specify the name of the virtual machine that has to be deleted.
GET disk/list? List Disk

POST disk/create Create Disk

  • vmdk: Name of the disk.

Plug-in tips

How will I uninstall the plug-in?

  1. On vCenter, open the CVS folder (Start > Program Files).
  2. Double click Uninstall CloudByte Virtual Service.

How will I upgrade the plug-in?

  1. Stop the CVS Server.
  2. To backup the database, copy the data folder (C:\Program Files (x86)\CloudByte Virtual Service\data ) to a location of your choice.
  3. Uninstall the plug-in.
  4. Install the latest version of the plug-in.
  5. If the Server is automatically started, stop the CVS Server.
  6. Replace the data folder with the one you backed up (in step 2).
  7. Start the CVS Server.

Note: CloudByte recommends you to backup daily the CVS plug-in database.

Limitations

  • In the case of restoring a virtual machine backup, you should ensure that the virtual machine is powered off.

Cloning, backup, and restore issues

  • The following limitations apply if an iSCSI datastore has multiple virtual machines:
    • When you restore a backup (snapshot) of a specific VM in a volume, instead of removing all subsequent snapshots on that VM, all subsequent snapshots of all VMs in the Volume are deleted.
      For instance, assume that the backups: backup1 of VM1, backup2 of VM2, backup3 of VM1. If you are restoring backup1, the backups backup2 and backup3 will be lost. It is recommended to create a single virtual machine on each iSCSI datastore.
    • Cloning a virtual machine clones all the other virtual machines on the datastore.
  • Rapid Clone utility functionality fails in the following scenarios:
    • When there are multiple disks attached to a virtual machine.
    • Rapid Cloning functionality fails if a virtual machine has files from local storage, for example, a CD/DVD drive or ISO on local ESX storage is attached to the virtual machine.
